The Customers API gives you read access to the buyer records in your store. Customer records are created when an order is placed and link buyers to their orders, giving you a consolidated view of your audience. Both list and single-resource endpoints require the `read:customers` scope.

---

## List Customers

Retrieve a paginated list of all customers in your store, sorted by `created_at` descending.

```
GET /customers
```

**Required scope:** `read:customers`

### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Maximum number of customers to return. Cannot exceed `100`.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of customers to skip before returning results. Use with `limit` to paginate.
</ParamField>

### Example Request

```bash
curl "https://iheiakmmrflybswwtknc.supabase.co/functions/v1/public-api/customers?limit=50&offset=0" \
  -H "Authorization: Bearer ts_live_xxxxxxxxxxxx"
```

### Example Response

```json
{
  "data": [
    {
      "id": "cus_01efgh",
      "first_name": "Jamie",
      "last_name": "Rivera",
      "email": "jamie.rivera@example.com",
      "phone": "+12025550198",
      "customer_type": "retail",
      "country": "US",
      "currency": "USD",
      "customer_code": "CUS-00412",
      "created_at": "2024-03-22T11:45:00Z"
    }
  ],
  "pagination": {
    "limit": 50,
    "offset": 0,
    "total": 1430
  }
}
```

## Retrieve a Customer

Fetch a single customer by their ID.

```
GET /customers/{id}
```

**Required scope:** `read:customers`

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the customer to retrieve.
</ParamField>

### Example Request

```bash
curl "https://iheiakmmrflybswwtknc.supabase.co/functions/v1/public-api/customers/cus_01efgh" \
  -H "Authorization: Bearer ts_live_xxxxxxxxxxxx"
```

### Example Response

```json
{
  "data": {
    "id": "cus_01efgh",
    "first_name": "Jamie",
    "last_name": "Rivera",
    "email": "jamie.rivera@example.com",
    "phone": "+12025550198",
    "customer_type": "retail",
    "country": "US",
    "currency": "USD",
    "customer_code": "CUS-00412",
    "created_at": "2024-03-22T11:45:00Z"
  }
}
```

---

## Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the customer, assigned by Tokensupply.
</ResponseField>

<ResponseField name="first_name" type="string">
  Customer's first name.
</ResponseField>

<ResponseField name="last_name" type="string">
  Customer's last name.
</ResponseField>

<ResponseField name="email" type="string">
  Customer's email address.
</ResponseField>

<ResponseField name="phone" type="string">
  Customer's phone number in E.164 format, if provided.
</ResponseField>

<ResponseField name="customer_type" type="string">
  Classification of the customer (e.g. `retail`, `wholesale`, `reseller`).
</ResponseField>

<ResponseField name="country" type="string">
  Two-letter ISO 3166-1 alpha-2 country code of the customer.
</ResponseField>

<ResponseField name="currency" type="string">
  Three-letter ISO 4217 currency code associated with the customer.
</ResponseField>

<ResponseField name="customer_code" type="string">
  A short human-readable reference code for the customer within your store.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the customer record was created.
</ResponseField>

<Tip>
  Use `customer_code` as a stable, readable reference when communicating with customers or cross-referencing records in external systems.
</Tip>
