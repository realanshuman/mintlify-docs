The Inventory API gives you full control over your store's fulfilment stock. Each inventory item holds a `value` — the actual key, code, or secret that gets delivered to a buyer. Because this is your own data, the API returns `value` in responses; however, it is never exposed over webhooks to protect your stock from interception. Use `read:inventory` to fetch items and `write:inventory` to add or update them.

---

## List Inventory

Retrieve a paginated list of all inventory items in your store, sorted by `created_at` descending.

```
GET /inventory
```

**Required scope:** `read:inventory`

### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Maximum number of items to return. Cannot exceed `100`.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of items to skip before returning results. Use with `limit` to paginate.
</ParamField>

### Example Request

```bash
curl "https://iheiakmmrflybswwtknc.supabase.co/functions/v1/public-api/inventory?limit=50&offset=0" \
  -H "Authorization: Bearer ts_live_xxxxxxxxxxxx"
```

### Example Response

```json
{
  "data": [
    {
      "id": "inv_01ijkl",
      "product_id": "prd_01abcd",
      "purchase_order_id": "po_01mnop",
      "order_id": "ord_01hxyz",
      "ref_id": "REF-9001",
      "value": "XXXX-YYYY-ZZZZ-1234",
      "status": "used",
      "expiry_date": "2025-12-31",
      "notes": "Batch A",
      "created_at": "2024-05-20T09:00:00Z"
    }
  ],
  "pagination": {
    "limit": 50,
    "offset": 0,
    "total": 320
  }
}
```

## Create an Inventory Item

Add a new inventory item (fulfilment key) to your store stock.

```
POST /inventory
```

**Required scope:** `write:inventory`

### Request Body

<ParamField body="product_id" type="string" required>
  The ID of the product this inventory item belongs to. Must belong to your store.
</ParamField>

<ParamField body="value" type="string" required>
  The fulfilment key, code, or secret to deliver to buyers. Stored securely and returned by the API, but never sent over webhooks.
</ParamField>

<ParamField body="ref_id" type="string">
  Your own reference identifier for this item (e.g. a batch number or external key ID).
</ParamField>

<ParamField body="purchase_order_id" type="string">
  The ID of the purchase order this item came from, if applicable.
</ParamField>

<ParamField body="expiry_date" type="string">
  Expiry date of the key in `YYYY-MM-DD` format. Leave blank if the key does not expire.
</ParamField>

<ParamField body="status" type="string">
  Initial status of the item. Defaults to `available` if omitted.
</ParamField>

<ParamField body="notes" type="string">
  Free-text notes for internal use (e.g. batch label, supplier reference).
</ParamField>

### Example Request

```bash
curl -X POST "https://iheiakmmrflybswwtknc.supabase.co/functions/v1/public-api/inventory" \
  -H "Authorization: Bearer ts_live_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "product_id": "prd_01abcd",
    "value": "XXXX-YYYY-ZZZZ-1234",
    "ref_id": "REF-9001",
    "expiry_date": "2025-12-31",
    "notes": "Batch A"
  }'
```

### Example Response

```json
{
  "data": {
    "id": "inv_01ijkl",
    "product_id": "prd_01abcd",
    "purchase_order_id": null,
    "order_id": null,
    "ref_id": "REF-9001",
    "value": "XXXX-YYYY-ZZZZ-1234",
    "status": "available",
    "expiry_date": "2025-12-31",
    "notes": "Batch A",
    "created_at": "2024-06-01T10:00:00Z"
  }
}
```

## Update an Inventory Item

Update one or more fields on an existing inventory item. At least one field must be provided.

```
PATCH /inventory/{id}
```

**Required scope:** `write:inventory`

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the inventory item to update.
</ParamField>

### Request Body

Provide at least one of the following fields:

<ParamField body="status" type="string">
  Updated status of the item (e.g. `available`, `used`, `voided`).
</ParamField>

<ParamField body="value" type="string">
  Replacement value for the fulfilment key or code.
</ParamField>

<ParamField body="expiry_date" type="string">
  Updated expiry date in `YYYY-MM-DD` format.
</ParamField>

<ParamField body="ref_id" type="string">
  Updated external reference ID.
</ParamField>

<ParamField body="notes" type="string">
  Updated free-text notes.
</ParamField>

### Example Request

```bash
curl -X PATCH "https://iheiakmmrflybswwtknc.supabase.co/functions/v1/public-api/inventory/inv_01ijkl" \
  -H "Authorization: Bearer ts_live_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "voided",
    "notes": "Replaced due to supplier recall"
  }'
```

### Example Response

```json
{
  "data": {
    "id": "inv_01ijkl",
    "product_id": "prd_01abcd",
    "purchase_order_id": null,
    "order_id": null,
    "ref_id": "REF-9001",
    "value": "XXXX-YYYY-ZZZZ-1234",
    "status": "voided",
    "expiry_date": "2025-12-31",
    "notes": "Replaced due to supplier recall",
    "created_at": "2024-05-20T09:00:00Z"
  }
}
```

<Warning>
  Sending a PATCH request with no recognised fields returns a `422 validation` error. Always include at least one of: `status`, `value`, `expiry_date`, `ref_id`, or `notes`.
</Warning>

## Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the inventory item, assigned by Tokensupply.
</ResponseField>

<ResponseField name="product_id" type="string">
  ID of the product this inventory item is associated with.
</ResponseField>

<ResponseField name="purchase_order_id" type="string">
  ID of the purchase order the item originated from, if any.
</ResponseField>

<ResponseField name="order_id" type="string">
  ID of the sales order this item was fulfilled against, if any.
</ResponseField>

<ResponseField name="ref_id" type="string">
  Your external reference ID for the item, if provided.
</ResponseField>

<ResponseField name="value" type="string">
  The fulfilment key, code, or secret. Returned by the API because it is your own data; never transmitted via webhooks.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the item (e.g. `available`, `used`, `voided`).
</ResponseField>

<ResponseField name="expiry_date" type="string">
  Expiry date of the key in `YYYY-MM-DD` format, or `null` if it does not expire.
</ResponseField>

<ResponseField name="notes" type="string">
  Free-text notes attached to the item.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the item was added to inventory.
</ResponseField>
