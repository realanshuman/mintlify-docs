Every webhook delivery contains a top-level `event` string that identifies what happened and a `data` object with the resource fields relevant to that event. This page lists every available event type grouped by resource, along with the fields present in each payload's `data` object. For details on delivery format, signature verification, and retry behaviour, see the [Webhooks guide](/api/webhooks).

<Note>
  Subscribe only to the event types your integration actually needs. You can update subscriptions at any time in **Settings â†’ Webhooks**.
</Note>

## Event catalog

<Accordion title="Orders">

Fired when an order changes state in any sales channel.

**Event types**

| Event | When it fires |
|---|---|
| `order.created` | A new order is placed |
| `order.completed` | An order is fulfilled and completed |
| `order.refunded` | A refund is issued on an order |
| `order.cancelled` | An order is cancelled |

**`data` payload fields**

| Field | Description |
|---|---|
| `id` | Internal order record ID |
| `order_id` | Tokensupply order identifier |
| `marketplace` | Sales channel the order came from |
| `marketplace_order_id` | Order ID in the originating marketplace |
| `status` | Current order status |
| `payment_status` | Payment state of the order |
| `product_id` | ID of the product ordered |
| `customer_id` | ID of the customer who placed the order |
| `quantity` | Number of units ordered |
| `amount` | Order total |
| `currency` | Three-letter currency code |
| `created_at` | ISO 8601 timestamp of order creation |

</Accordion>

<Accordion title="Inventory">

Fired when inventory items are added, sold, or when stock for a product runs low.

**Event types**

| Event | When it fires |
|---|---|
| `inventory.created` | New inventory items are added to a product |
| `inventory.sold` | An inventory item is delivered to a customer |
| `inventory.low_stock` | Available stock for a product drops to 5 or fewer units |

**`data` payload fields â€” `inventory.created` and `inventory.sold`**

| Field | Description |
|---|---|
| `id` | Inventory item ID |
| `ref_id` | External reference identifier |
| `product_id` | ID of the associated product |
| `purchase_order_id` | ID of the purchase order that sourced this item |
| `status` | Current item status |
| `expiry_date` | Expiry date of the item, if applicable |
| `created_at` | ISO 8601 timestamp of item creation |

**`data` payload fields â€” `inventory.low_stock`**

| Field | Description |
|---|---|
| `product_id` | ID of the product with low stock |
| `available` | Current number of available units |
| `threshold` | Stock level that triggered the alert |

<Warning>
  The raw inventory value (the key or secret you sell) is **never** included in any webhook payload. Webhook data is safe to log and forward without risk of exposing customer credentials.
</Warning>

</Accordion>

<Accordion title="Integrations">

Fired when a marketplace or sales channel integration is connected or disconnected.

**Event types**

| Event | When it fires |
|---|---|
| `integration.connected` | A marketplace integration is successfully connected |
| `integration.disconnected` | A marketplace integration is removed or loses access |

**`data` payload fields**

| Field | Description |
|---|---|
| `id` | Integration record ID |
| `marketplace` | Name of the marketplace or sales channel |
| `is_active` | Whether the integration is currently active |
| `created_at` | ISO 8601 timestamp of integration creation |

<Note>
  Integration credentials (API keys, OAuth tokens, etc.) are **never** included in webhook payloads.
</Note>

</Accordion>

<Accordion title="Test">

Use the `ping` event to confirm that your endpoint is reachable and your signature verification logic is working correctly.

**Event types**

| Event | When it fires |
|---|---|
| `ping` | Manually, when you click **Send test** on an endpoint in Settings â†’ Webhooks |

The `ping` event delivers a minimal payload with no meaningful `data` fields. Use it to validate your endpoint setup end-to-end before going live with real events.

</Accordion>
