The Vendors API gives you read access to the vendor records associated with your store. Vendors are the suppliers or counterparties you source inventory from. Both list and single-resource endpoints require the `read:vendors` scope. To protect sensitive financial information, bank account details, PAN numbers, and similar data are intentionally excluded from all API responses — access those details through the Tokensupply dashboard only.

---

## List Vendors

Retrieve a paginated list of all vendors associated with your store, sorted by `created_at` descending.

```
GET /vendors
```

**Required scope:** `read:vendors`

### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Maximum number of vendors to return. Cannot exceed `100`.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of vendors to skip before returning results. Use with `limit` to paginate.
</ParamField>

### Example Request

```bash
curl "https://iheiakmmrflybswwtknc.supabase.co/functions/v1/public-api/vendors?limit=50&offset=0" \
  -H "Authorization: Bearer ts_live_xxxxxxxxxxxx"
```

### Example Response

```json
{
  "data": [
    {
      "id": "ven_01yzab",
      "name": "Digital Keys Wholesale Ltd",
      "email": "supply@digitalkeyswholesale.com",
      "phone": "+447911123456",
      "seller_type": "distributor",

}
```

## Retrieve a Vendor

Fetch a single vendor by their ID.

```
GET /vendors/{id}
```

**Required scope:** `read:vendors`

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the vendor to retrieve.
</ParamField>

### Example Request

```bash
curl "https://iheiakmmrflybswwtknc.supabase.co/functions/v1/public-api/vendors/ven_01yzab" \
  -H "Authorization: Bearer ts_live_xxxxxxxxxxxx"
```

### Example Response

```json
{
  "data": {
    "id": "ven_01yzab",
    "name": "Digital Keys Wholesale Ltd",
    "email": "supply@digitalkeyswholesale.com",
    "phone": "+447911123456",
    "seller_type": "distributor",
    "account_type": "business",
    "status": "active",
    "country": "GB",
    "currency": "GBP",
    "seller_code": "DKW-001",
    "created_at": "2023-09-01T08:00:00Z"
  }
}
```

## Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the vendor, assigned by Tokensupply.
</ResponseField>

<ResponseField name="name" type="string">
  Display name of the vendor or supplier.
</ResponseField>

<ResponseField name="email" type="string">
  Contact email address for the vendor.
</ResponseField>

<ResponseField name="phone" type="string">
  Contact phone number for the vendor in E.164 format.
</ResponseField>

<ResponseField name="seller_type" type="string">
  The vendor's role in the supply chain (e.g. `distributor`, `manufacturer`, `reseller`).
</ResponseField>

<ResponseField name="account_type" type="string">
  Whether the vendor is a `business` or `individual` account.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the vendor relationship (e.g. `active`, `inactive`, `suspended`).
</ResponseField>

<ResponseField name="country" type="string">
  Two-letter ISO 3166-1 alpha-2 country code of the vendor.
</ResponseField>

<ResponseField name="currency" type="string">
  Three-letter ISO 4217 currency code used for transactions with this vendor.
</ResponseField>

<ResponseField name="seller_code" type="string">
  A short reference code for the vendor, used internally within Tokensupply.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the vendor record was created.
</ResponseField>

<Warning>
  Bank account numbers, PAN details, and other sensitive financial information are intentionally excluded from API responses. Retrieve those details through the Tokensupply dashboard.
</Warning>
