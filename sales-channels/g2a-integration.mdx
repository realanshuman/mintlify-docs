G2A is one of the largest marketplaces for digital game keys and software. TokenSupply ships a **full two-way integration**: catalogue search, listing management, automatic price and stock sync, checkout reservations, auto-delivery, and refund handling — all wired into your inventory and orders.

This guide walks you through everything end-to-end — from prerequisites to going live.

## Before you start

Make sure you have:

<Note>
  If you sell B2B keys, confirm your G2A account is enabled for the **Business price tier** — otherwise the business price field is ignored by G2A.
</Note>

## Step 1 — Get your G2A API credentials

G2A issues credentials from the Seller Panel.

<Steps>
  <Step title="Sign in to the Seller Panel">
    Go to [sales.g2a.com](https://sales.g2a.com) and sign in.
  </Step>
  <Step title="Open the API section">
    Navigate to **My Account → API**.
  </Step>
  <Step title="Generate a new API key">
    Click **Generate new API key** (or copy your existing one). You'll receive four values — copy them somewhere safe, the secret is shown only once.

    | Field | Description |
    | --- | --- |
    | **Client ID** | Public identifier (UUID-style) |
    | **Client Secret** | Private token — never share |
    | **API URL** | Production: `https://api.g2a.com` · Sandbox: `https://sandboxapi.g2a.com` |
    | **Token URL** | Production: `https://api.g2a.com/oauth/token` · Sandbox: `https://sandboxapi.g2a.com/oauth/token` |
  </Step>
  <Step title="Enable the required scopes">
    In the same panel, enable: **Marketplace API**, **Import API** (for reservations/checkout), and **Refunds**.
  </Step>
</Steps>

<Warning>
  Regenerating the secret rotates it immediately. Existing reservations stay intact, but new auth tokens will be issued on the next sync cycle.
</Warning>

## Step 2 — Connect G2A to TokenSupply

<Steps>
  <Step title="Open the connector">
    In TokenSupply, go to **Integrations → G2A → Connect**.
  </Step>
  <Step title="Paste your credentials">
    Enter your **Client ID**, **Client Secret**, **API URL**, and **Token URL**.
  </Step>
  <Step title="Choose an environment">
    Pick **Sandbox** for the first hour of testing, then switch to **Production**.
  </Step>
  <Step title="Test the connection">
    Click **Test Connection**. TokenSupply requests an OAuth token from G2A — a green tick means your credentials are valid.
  </Step>
  <Step title="Accept the contract terms">
    When prompted, accept the G2A contract terms. This authorises TokenSupply to act as your fulfilment endpoint for the G2A Import API.
  </Step>
  <Step title="Save">
    The integration card now shows **Connected** with your seller ID.
  </Step>
</Steps>

### Troubleshooting connection errors

| Error | Meaning | Fix |
| --- | --- | --- |
| `invalid_client` | Client ID/Secret mismatch | Re-copy from the Seller Panel |
| `unauthorized_client` | API access not enabled | Open a ticket with G2A support |
| `403 Forbidden` on `/products` | Marketplace scope missing | Regenerate key with Marketplace scope |
| Network timeout | Wrong API URL / sandbox vs. prod mismatch | Re-check the URL pair |

## Step 3 — Configure the Import API in G2A

This is the most important step for checkout to work. The **Import API** is how G2A talks to TokenSupply when a buyer purchases your item.

<Steps>
  <Step title="Open your G2A Merchant Dashboard">
    Log in at [g2a.com/merchant-area](https://www.g2a.com/en/merchant-area).
  </Step>
  <Step title="Find the Contract API settings">
    Go to **Settings → Contract API** (called **Import API Settings** on some dashboards).
  </Step>
  <Step title="Paste the three URLs from TokenSupply">
    On your TokenSupply G2A integration page, each URL has a copy button next to it. Paste them straight in:

    | Field in G2A | What to paste | What it does |
    | --- | --- | --- |
    | **Token URL** | TokenSupply URL \+ `/oauth/token` | G2A logs in here before every request. Issues a 1-hour JWT. |
    | **API URL** | TokenSupply base URL (ends in `/g2a-contract`) | G2A sends reservations, orders, and stock checks here. |
    | **Health Check** | TokenSupply base URL \+ `/health` | G2A pings this to confirm your endpoint is alive. |
  </Step>
</Steps>

Once saved, G2A automatically routes buyer traffic to TokenSupply.

### What G2A calls and when

| Endpoint | When it's called | What TokenSupply does |
| --- | --- | --- |
| `POST /check` | Buyer views your product page | Confirms stock is available |
| `POST /reservation` | Buyer clicks **Buy Now** | Locks the key for 15 min. Status → **Reserved** |
| `PUT /reservation/{id}` | Reservation is about to expire | Extends the hold |
| `DELETE /reservation/{id}` | Buyer abandons cart | Releases the key. Status → **Available** |
| `POST /order` | Buyer completes payment (Auto Deliver ON) | Delivers the key instantly. Status → **Sold** |
| `GET /inventory/{orderId}` | Buyer or G2A re-reads the key | Returns the delivered key payload again |

<Info>
  **Auto Deliver = ON** — buyer gets the key the instant payment clears. **Auto Deliver = OFF** — key stays **Reserved** until you manually deliver it from the Transactions page.
</Info>

## Step 4 — Set up the Export API

The Import API is G2A calling TokenSupply. The **Export API** is the other direction — TokenSupply calling G2A — and powers everything outbound.

<Steps>
  <Step title="Grab Export credentials">
    In the same **Seller Panel → API** section, find the **Export API** (or Marketplace API) credentials.
  </Step>
  <Step title="Paste them into TokenSupply">
    Open the **Export API** section on your TokenSupply G2A integration page and paste the **Export Client ID** and **Export Client Secret**.
  </Step>
</Steps>

<Tip>
  If G2A shows only one credential pair, it usually works for both Import and Export. Paste the same pair into both sections in TokenSupply.
</Tip>

### What the Export API powers

| Feature | What TokenSupply sends |
| --- | --- |
| **Listings sync** | Price, stock count, and active/inactive status for every mapped product |
| **Orders sync** | Pulls new G2A orders into your Transactions page |
| **Refund sync** | Pulls refunds and cancellations, updates inventory automatically |
| **Catalogue search** | Searches G2A's full product database when mapping new products |

<Warning>
  Without Export API credentials, your G2A connection only works one way — buyers can check out, but nothing appears in your dashboard.
</Warning>

## Mapping internal products to G2A offers

A mapping ties one of your internal products to one G2A product ID. Until a product is mapped, it cannot be listed.

<Steps>
  <Step title="Open the Listings tab">
    On the integration page, click the **Listings** tab.
  </Step>
  <Step title="Add a listing">
    Click **Add Listing** → pick an internal product → search and select the G2A product.
  </Step>
  <Step title="Configure the listing">
    | Field | Description |
    | --- | --- |
    | **Retail price** | Your B2C selling price (EUR by default; G2A converts on display) |
    | **Business price** | Optional B2B tier — must be ≤ retail |
    | **Declared stock** | Auto-capped at your **Available** key count, so you cannot oversell |
    | **Delivery time** | One of G2A's allowed values (e.g. 0–30 min, 1–8 hours, 1 day). Auto-deliver listings should use the shortest band |
    | **Active** | Toggles the offer on/off on G2A without deleting the mapping |
    | **Auto Deliver** | Delivers the key instantly when G2A confirms payment |
  </Step>
  <Step title="Save">
    The listing appears in G2A within a minute on the next sync.
  </Step>
</Steps>

<Tip>
  Each listing shows **competing dropshipping offers** from other G2A sellers on the same SKU, so you can price competitively at a glance.
</Tip>

## Auto Sync

Manual "Sync now" buttons have been intentionally removed. Syncs run automatically through scheduled jobs (`pg_cron`) so the cadence is deterministic and stays within G2A's rate limits.

| Sync type | What it does |
| --- | --- |
| **Listings sync** | Pushes price, stock, and active status changes to G2A |
| **Orders sync** | Pulls newly placed G2A orders into the Transactions module |
| **Refund sync** | Pulls refunds and cancellations, updates inventory accordingly |

Enable **Auto Sync** on the integration page and pick an interval that matches your volume. The UI shows recommended values — faster intervals run tighter against G2A's quota.

## Checkout, reservations, and auto-delivery

When a buyer adds your offer to their G2A cart, G2A calls TokenSupply's Import API at the `g2a-contract` edge function. The contract follows G2A's documented shape **strictly** — extra fields cause checkout to silently fail.

| Endpoint | Behaviour |
| --- | --- |
| `POST /reservation` | Reserves N keys. Returns only `reservation_id` and a `stock` array. Inventory items flip to **Reserved** |
| `PUT /reservation/{id}` | Extends the reservation window |
| `DELETE /reservation/{id}` | Releases the reservation. Items go back to **Available** |
| `POST /order` (auto-deliver) | G2A confirms purchase. TokenSupply marks inventory **Sold**, completes the transaction, returns the key payload in the same response |

<Info>
  Price and currency are read from G2A's `additional_data` field on the inbound payload — no slow live lookup per reservation.
</Info>

<Warning>
  If a buyer ever sees **"cart updated / out of stock"** during checkout, it almost always means an upstream change broke the strict reservation response. The current contract is locked to G2A's spec to prevent this.
</Warning>

## Refunds and cancellations

G2A pushes refunds and cancellations through the **Refunds API**. The refund-sync cron pulls them every minute and applies these rules:

- A refund on a delivered (**Sold**) key flips the inventory item to **Invalid** — refunded keys cannot be resold (G2A considers them burned).
- A cancellation on a reserved-but-not-delivered item releases the key to **Available**.
- A safety cron catches any **Reserved** item linked to a refunded/failed/cancelled transaction older than 15 minutes and releases it.
- A database trigger guarantees `order_id` is always nulled when an item returns to **Available** or **Invalid** — no code path can leave a stale order link behind.

Audit any item's history from **Inventory → key → Activity log**.

## Rate limits

G2A enforces strict request limits per endpoint.

| Endpoint group | Limit |
| --- | --- |
| Listings (price/stock) | Burstable. Sustained traffic above ~10 req/s returns **ERR429** |
| Product search | Much tighter — the UI debounces keystrokes for you |
| Reservation/order | Driven by G2A, no client-side limit needed |

If you see **ERR429** in the activity log, lengthen your Auto Sync interval. TokenSupply's defaults are tuned to stay within published quotas at typical seller volumes.

## Troubleshooting

| Symptom | First place to check |
| --- | --- |
| Listing not appearing on G2A | **Listings** tab → **Active** toggle → last sync time |
| Buyer cart says "out of stock" | **Inventory** → mapped product → **Available** count |
| Order didn't auto-deliver | **Transactions** → row status → **Auto Deliver** toggle on the listing |
| Repeated **ERR429** in activity log | **Auto Sync** interval is too aggressive |
| Refunded key still shows in a Sold transaction | Refund-sync cron — the key should be **Invalid**, not **Available** |

For anything else, open a ticket from **Help Centre → Contact Support** with the G2A transaction ID — we can match it to the inbound contract payload immediately.
